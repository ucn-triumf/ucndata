# ucndata — Unit Test Plan

| | |
|---|---|
| **Status** | Plan (to be executed with pytest) |
| **Target** | > 90% statement coverage of the `ucndata` package |
| **Constraint** | No live experimental data — all input is synthetic and generated by the test suite itself |
| **Package version** | 4.5.5 |

This document is the blueprint for the `tests/` suite. It specifies the
strategy, fixtures, per-module test cases, known defects, and the path to the
coverage target. Read it top-to-bottom before any test code is written.

---

## 1. Objectives

1. Verify every public class, method, and property in `ucndata` behaves as
   documented for the "golden path" of normal usage.
2. Exercise edge cases and error paths (bad input, missing trees, empty data,
   negative/out-of-range indices, filtered cycles, zero-length periods).
3. Achieve **> 90% statement coverage**, measured with `pytest-cov`.
4. Run fast and hermetically: no network, no `/data3/ucn/root_files`, no real
   `.root` files from the experiment. Every test must be reproducible on any
   machine that has ROOT + rootloader installed.
5. Surface and document existing bugs (see [section 7](#7-known-bugs--xfail-register))
   rather than hide them.

---

## 2. Constraints & Strategy — "No Live Data"

`ucndata` is fundamentally a reader of ROOT files produced by `midas2root`.
"No live data" therefore means: the suite must **not** depend on any real run
file. It does **not** mean ROOT must be avoided — ROOT and `rootloader` are
genuine, installed dependencies (confirmed available: ROOT, rootloader,
pytest 9.x, pytest-cov, Python 3.12).

Three complementary test layers are used:

### Layer A — Pure-Python unit tests (no ROOT)

Modules with no ROOT dependency are tested directly:

- `applylist.py`
- `datetime.py`
- `exceptions.py` -- not tested
- `constants.py` -- not tested

These are the cheapest tests and should run first.

### Layer B — Synthetic ROOT file integration tests

A small, fully deterministic ROOT file is generated by `conftest.py` using raw
PyROOT (see [section 5](#5-fixtures-conftestpy)). Because the suite builds the
file, every value (cycle count, period durations, hit timestamps, beam current)
is known in advance, so assertions can check exact numbers. This drives the
real code paths of `ucnrun`, `ucncycle`, `ucnperiod`, `crun`, `tsubfile`, and
`ttreeslow` end-to-end.

### Layer C — Empty-object construction tests

`ucnrun.__init__` returns early when called with `run=None`. This hook is used
to build a bare object and hand-populate only the attributes a given method
needs. This isolates branch-heavy methods (`set_cycle_times`,
`_modify_ptiming`, `check_data`, `_get_cycle_param`) so each logical branch can
be tested without building a bespoke ROOT file per variant.

> **Rationale** for a real synthetic ROOT file over mocking `rootloader.tfile`:
> the hit-counting paths (`_get_nhits`, `get_hits_histogram`, `trigger_edge`)
> depend on real `RDataFrame.Histo1D` behaviour. A faithful mock would be
> larger and less trustworthy than a 3-cycle synthetic file.

## 3. Tooling & Configuration

**Global test environment requirements:**

- matplotlib **must** use the `Agg` backend. Set in `conftest.py` *before* any
  `ucndata` import: `import matplotlib; matplotlib.use("Agg")`.
- `__repr__` of every ucn object calls `os.get_terminal_size()`, which raises
  `OSError` when stdout is not a tty (i.e. under pytest capture). A session
  fixture must patch this — see [section 5.4](#54-support-fixtures).
- ROOT runs in batch mode (`ucnrun` already sets `ROOT.gROOT.SetBatch(1)`).

## 5. Fixtures (`conftest.py`)

### 5.1 Synthetic ROOT file builder (`_root_builder.py`)

A helper module exposing `build_run_file(path, **opts)` that writes a ROOT file
with the trees `ucndata` expects. Use raw PyROOT (`ROOT.TFile`, `ROOT.TTree`,
`TBranch`) rather than `RDataFrame.Snapshot`, because:

- header branch names must contain spaces (e.g. `"Run Number"`) to exercise
  `ucnrun.__init__`'s name-normalisation;
- `RunTransitions_*` trees need `std::vector<int>` branches (valve states);
- raw TTree gives full control over branch names and dtypes.

The builder must accept options so the same code produces every file variant in
[section 5.3](#53-file-variants-session-scoped-fixtures) (sequencer on/off,
transitions present/absent, He3/Li6 offset).

### 5.2 Schema of the synthetic trees

The "good" file models a 3-cycle run, 3 periods per cycle, periods of duration
`[20, 30, 50]` s, cycle length 100 s, run start epoch `t0`.

#### `header` — 1 row, string/scalar branches **with spaces in names**

| Branch | Value |
|---|---|
| `Run Number` | `2684` |
| `Run Title` | `"synthetic run"` |
| `Start Time` | `"2024-06-01 12:00:00"` |
| `Stop Time` | `"2024-06-01 12:05:00"` |
| `Shifters` | `"tester"` |
| `Comment` | `"synthetic"` |
| `Experiment Number` | `"2025-01"` |

#### `CycleParamTree` — 1 row

`nPeriods=3`, `nSuperCyc=1`, `enable=1`, `infCyclesEnable=0`

#### `RunTransitions_He3` / `RunTransitions_Li6` — 3 rows, one per cycle

| Branch | Value |
|---|---|
| `cycleIndex` | `[0, 1, 2]` |
| `superCycleIndex` | `[0, 0, 0]` |
| `cycleStartTime` | `[t0, t0+100, t0+200]` |
| `cyclePeriod0EndTime` | `cycleStartTime + 20` |
| `cyclePeriod1EndTime` | `cycleStartTime + 50` |
| `cyclePeriod2EndTime` | `cycleStartTime + 100` |
| `valveStatePeriod0/1/2` | `std::vector<int>`, at least one valve set to `1` |

He3 and Li6 `cycleStartTime` are **identical** in the good file (matched mode).
At least one valve must be `1` so `ucncycle.check_data`'s valve check passes.

#### `SequencerTree` — time-indexed, index branch `"timestamp"`

- `sequencerEnabled` = `1` throughout
- `cycleStarted` = transitions to `1` at each cycle start
- `inCycle` = `1` inside cycles, `0` between

#### `BeamlineEpics` — time-indexed, branch `"timestamp"`, ~1 row / 5 s

- `B1_FOIL_ADJCUR` = constant ~1.0 uA (above `beam_min_current` 0.1)
- `B1V_KSM_PREDCUR` = constant
- `B1V_KSM_BONPRD` = `1`
- `B1V_KSM_RDBEAMON_VAL1` = beam-on bucket count (so `beam_on_s` ≈ 60 s)
- `B1V_KSM_RDBEAMOFF_VAL1` = beam-off bucket count

#### `LNDDetectorTree` — time-indexed, non-empty (required by `check_data`)

#### `UCNHits_He3` / `UCNHits_Li-6` — event-level, index `tUnixTimePrecise`

- `tUnixTimePrecise` — a **known** set of hit timestamps spread across the run,
  with a controlled count inside each period so `_get_nhits` results can be
  asserted exactly. Some timestamps `< 15e8` to exercise the bad-timestamp
  filter.
- `tIsUCN` — mix of `0` and `1` (so the `ucn_only` filter is meaningful and
  `check_data`'s "any UCN hits" check passes)
- `tChannel` — `0..9` for detector hits; channel `15` used for chopper frames
- `tChargeL` — positive values `< 5e3` (for `plot_psd`)
- `tPSD` — `0..1` values

> `"UCNHits_Li-6"` keeps the dash so `__init__`'s dash-stripping rename
> (→ `"UCNHits_Li6"`) is exercised.

#### EPICS trees for `ttreeslow`

At minimum `BeamlineEpics` plus one extra tree from `ucnbase.EPICS_TREES`
(e.g. `"UCN2Epics"`) with distinct column names, so `ttreeslow` genuinely
merges multiple trees.

> All timestamps are plain ints/floats in epoch seconds. Every count and
> duration is chosen so tests assert exact integers, not ranges.

### 5.3 File variants (session-scoped fixtures)

| Fixture | Description |
|---|---|
| `good_run_file` | Full schema above. Drives most of Layer B. |
| `no_sequencer_file` | `SequencerTree.sequencerEnabled` all `0` → whole run collapses to a single cycle. |
| `no_transitions_file` | `RunTransitions_*` trees omitted → `_get_cycle_param` emits `MissingDataWarning`, `ncycles = nperiods = 1`. |
| `mismatched_file` | He3 vs Li6 `cycleStartTime` differ by > 20 s → `matched` mode raises `CycleError`; detection falls through to `li6`/`he3`. |
| `no_slow_trees_file` | `SLOW_TREES` absent → `set_cycle_times` raises `MissingDataError`; `check_data` returns `False`. |
| `chopper_file` | `UCNHits_Li6` populated with channel-15 frame markers at known timestamps → drives `test_chopper.py`. |

Each variant is built once per session into `tmp_path_factory` and the path is
returned by the fixture.

## 6. Module-by-Module Test Specifications

### 6.1 `test_applylist.py` — Layer A (pure Python)

**Core:**

- construction from range/list/np.ndarray; `isinstance(x, list)`.
- element attribute access: `applylist([obj1, obj2]).attr` → applylist of
  attrs; recursive on nested applylist.
- element method call via `__call__`: `applylist([s1, s2]).upper()`.
- `apply(fn)` returns new list, leaves original unchanged.
- `apply(fn, inplace=True)` returns `None` and mutates.
- `apply` recurses into nested applylist.
- arithmetic: `+ - * / // % **` element-wise vs scalar and vs sequence.
- comparisons: `== != < > <= >=` return applylist of numpy bools.
- unary ops: `- + abs invert round floor ceil trunc`.
- bitwise: `& | ^`.
- `__getitem__` plain int → element; plain slice → applylist.
- `__getitem__` boolean np.ndarray → selection.
- `__getitem__` integer np.ndarray → reorder.
- `__getitem__` with list/tuple of indices.
- `astype(int)` converts in place.
- `transpose()` of a 2-D applylist.

**Edge:**

- attribute access on applylist of scalar numpy types raises `AttributeError`
  (the `isinstance(self[0], (int, float, ...))` guard).
- names containing `__array_` raise `AttributeError` (numpy protocol guard).
- attribute access on an **empty** applylist raises `IndexError` (`self[0]`).
- real list attributes (`.copy`, `.append`) still resolve normally.

### 6.2 `test_datetime.py` — Layer A

**Core:**

- `to_datetime` on an int array → tz-aware `DatetimeIndex` in `ucnbase.timezone`
  (`America/Vancouver`).
- `to_datetime` on a DataFrame converts the index, preserves columns and index
  name.
- `from_datetime` on a tz-aware Series → integer epoch seconds.
- `from_datetime` on a DataFrame converts index, preserves name.
- **round trip**: `from_datetime(to_datetime(x)) == x` for an epoch int array
  and for a DataFrame index.

**Edge:**

- `from_datetime` on a tz-**naive** datetime input hits the `tz_localize`
  branch (TypeError fallback) and still succeeds.
- sub-second timestamps: confirm floor-division to whole seconds.
- empty input array.

### 6.3 `test_tsubfile.py` — Layer B

Build a `rootloader.tfile` from the good file, wrap a slice with `tsubfile`.

**Core:**

- `tsubfile(tfileobj, start, stop)` exposes the same keys as the parent.
- accessing a time-indexed DataFrame returns only rows in `[start, stop]`.
- accessing a time-indexed `ttree` returns a `.loc`-sliced ttree.
- `_start` / `_stop` retrievable via item and attribute access.
- repeated access returns the **cached** object (identity check on `_items`).
- `__getattr__` falls back to real attributes for non-keys.

**Edge:**

- a non-time-indexed DataFrame is returned unfiltered.
- an object whose index name lacks `time` is returned unfiltered.
- `start == stop` yields an empty (but valid) slice.
- start/stop entirely outside the data range → empty result, no error.

### 6.4 `test_ttreeslow.py` — Layer B

Construct with the parent `ucnrun` (which builds `run.epics`).

**Core:**

- `columns` lists the union of all member-tree columns.
- `treenames` returns unique tree names.
- `__getattr__` on a known column routes to the owning tree.
- `__getitem__` on a known column returns the column data.
- `mean`/`min`/`max`/`rms`/`std` concatenate per-tree results into one Series.
- `index` / `index_name` / `filters` return per-tree dicts.
- copy constructor `ttreeslow(existing)` copies `_columns`/`_treenames`.
- copy constructor with explicit `parent=` overrides the parent.
- `reset()` returns a fresh `ttreeslow`.
- `set_index(col)` propagates to every member tree.
- `set_filter(expr)` propagates when `inplace=True` (default).

**Edge:**

- constructing from a tree list with `parent=None` raises `RuntimeError`.
- `set_filter(expr, inplace=False)` raises `RuntimeError`.
- `__getattr__` on an unknown name falls through to `getattr(object, name)`.
- `hist1d(column)` returns a `th1` histogram for the named column.
- `hist1d()` with a single-column `ttreeslow` infers the column automatically.

### 6.5 `test_ucnbase.py` — Layer B + C

`ucnbase` is abstract-ish; test it through a real `ucnrun` and a `ucncycle`.

**Core:**

- `apply(fn)` over a run returns an applylist of per-cycle results.
- `get_hits_array('Li6')` / `('He3')` return a numpy array of timestamps;
  length equals the post-filter hit count.
- `get_hits_histogram('He3')` returns a `th1`; `bin_ms` changes bin count;
  `as_datetime=True` converts `hist.x` to datetimes.
- `get_hits_histogram` on a cycle/period trims `hist` to the cycle/period
  window (the `frame_start`/`period_start`/`cycle_start` branch ladder).
- properties `beam1a_current_uA`, `beam1u_current_uA` return Series; `beam1u`
  equals `PREDCUR * BONPRD`.
- `beam_on_s` / `beam_off_s` return a per-cycle Series; for a single-cycle
  object return a float; values match
  `RDBEAMON/OFF_VAL1 * beam_bucket_duration_s`.
- `trigger_edge('Li6', thresh)` returns rising-edge times; `rising=False`
  returns falling edges.
- `_get_beam_duration` selects the cycle when called on a single cycle.

**Edge:**

- `get_hits_array` / `get_hits_histogram` with an unknown detector name raise
  `KeyError`.
- `trigger_edge` with a threshold above every bin raises `RuntimeError`
  ("No position from trigger edge found").
- `trigger_edge` on an empty histogram raises `RuntimeError`
  ("Histogram has no data").
- `_get_beam_duration` when `BeamlineEpics` is absent raises `MissingDataError`.
- `beam1a_current_uA` works whether `BeamlineEpics` is a DataFrame or a ttree
  (cover both branches).

**Plotting** (marker: `plotting`):

- `inspect()` on a run returns an axes array; runs with `xmode` in
  `{'duration', 'datetime', 'epoch'}` and with `slow=` a str and a list.
- `inspect()` with bad `xmode` raises `RuntimeError`.
- `inspect()` with `slow=` an unknown column raises `KeyError`.
- `inspect()` on a **period** raises `NotImplementedError` (the custom one).
- `plot_psd('Li6')` runs without error; `cut=(x, y)` draws the box.

### 6.6 `test_ucnrun.py` — Layer B + C

**Construction / parsing:**

- `ucnrun(path_str)` loads; header fields land as attributes (`run_number`
  int, `run_title`, `start_time`, `year`, `month`).
- branch names with spaces/dashes are normalised in `tfile`.
- `DATAFRAME_TREES` are converted to DataFrames with a reset index.
- `ucnrun(int)` builds the path from `datadir`; assert the constructed path
  (point `datadir` at the tmp dir, or assert on the `IOError` message for a
  missing run number).
- `ucnrun(None)` returns a bare object (Layer C hook).
- `ucnrun(unsupported_type)` raises `TypeError`.
- `epics` is a `ttreeslow`; `_run` points to self; caches initialised
  (`_cycledict`, `_hits_hist`, `_nhits`).

**`_get_cycle_param`:**

- good file: `nperiods`, `nsupercyc`, `enable`, `inf_cyc_enable`, `ncycles`,
  `valve_states`, `period_end_times`, `period_durations_s` all correct; assert
  exact period durations `[20, 30, 50]`.
- `no_transitions_file`: emits `MissingDataWarning`; `ncycles = nperiods = 1`;
  `cycle`/`supercycle` Series created.

**`set_cycle_times`** (parametrize over modes; use Layer C where convenient):

- `matched`: good file → start/stop/duration/offset columns; offset is
  He3 − Li6 (0 in the good file).
- `li6` and `he3`: start times from the respective transition tree.
- `sequencer`: starts/stops from `inCycle` diff.
- no-sequencer file: whole run collapses to one cycle.
- mismatched file in `matched` mode raises `CycleError` (offset > 20 s).
- missing slow trees raises `MissingDataError` (infinite `run_stop`).
- bad mode string raises `RuntimeError`.
- called on a single cycle returns the one-row cycle frame.
- back-fills `period_end_times`/`period_durations_s` when absent.

**`__getitem__` / slicing / iteration:**

- `run[0]` → ucncycle; `run[-1]` → last cycle (negative wrap).
- `run[i > ncycles]` raises `IndexError`.
- `run[0, 0]` → ucnperiod; `run[:, 0]` → applylist of period-0 objects.
- `run[1:3]` → applylist of cycles; `run[:]` → all cycles.
- `run[0, :]` → all periods of cycle 0.
- bad index type raises `IndexError`.
- `len(run) == ncycles`.
- iteration yields every cycle; with a filter set, filtered cycles skipped.
- `__repr__` produces a non-empty string (terminal-size fixture active); bare
  `ucnrun(None)` repr returns `"ucnrun()"`.

**`get_cycle`:**

- `get_cycle(0)` returns and caches a ucncycle (identity on 2nd call).
- `get_cycle()` / negative arg returns an applylist of all cycles.

**`set_cycle_filter` / `gen_cycle_filter`:**

- `set_cycle_filter(bool_array)` stores the filter and propagates to cached
  cycles.
- wrong-length filter raises `RuntimeError`.
- `set_cycle_filter(None)` clears (acts as all-True).
- filtered slice `run[:]` skips rejected cycles; unfiltered `get_cycle()` still
  returns all.
- `gen_cycle_filter()` returns a bool array of length `ncycles`.

**`check_data`:**

- good file → `True`.
- missing a `SLOW_TREES` tree → `False` (and `raise_error=True` →
  `MissingDataError`).
- empty slow tree → `False`.
- hit tree with no UCN hits → `False`.

**`keyfilter`:**

- rejects names containing `rate`, `charge`, `v1725`, `v1720`, `v792`, `iv2`,
  `iv3`, `edge_diff`, `pulse_widths`.
- accepts ordinary names (`BeamlineEpics`, `UCNHits_He3`).

**`_get_nhits`:**

- run-level (`cycle=None`) returns total tree size.
- cycle-level and period-level counts match the known injected counts.
- `bin_ms=0` (raw) vs `bin_ms>0` (binned) paths both covered.
- changing `bin_ms` regenerates the cached histogram.
- zero-length period reports 0 hits.

**`_modify_ptiming` / period timing:**

- shifting a period start moves `period_end_times`; durations recomputed.
- shifting a stop is clamped to the cycle end.
- period 0 start shift moves the cycle start.
- modifying timing clears `_nhits` and drops the affected cached cycle.
- recursion path for adjacent zero-length periods.

### 6.7 `test_ucncycle.py` — Layer B + C

**Core:**

- `run[0]` is a `ucncycle`; `cycle`, `supercycle`, `cycle_start`, `cycle_stop`,
  `cycle_dur` correct.
- `cycle_param.period_durations_s` / `period_end_times` trimmed to this cycle.
- `tfile` is a `tsubfile`; `epics` is a `ttreeslow` bound to the cycle.
- `len(cycle) == nperiods`.
- iteration yields each `ucnperiod`.
- `cycle[0]` → ucnperiod; `cycle[-1]` negative wrap; `cycle[:]` → applylist.
- `cycle[i > nperiods]` raises `IndexError`.
- non-1-D index raises `IndexError`.
- `get_period(0)` caches (identity on repeat); `get_period()` → all.
- `__repr__` non-empty.

**`check_data`** (each branch, Layer C with hand-set attributes):

- all checks pass → `True`.
- no 1A beam data → `BeamError` / `False` / warn depending on flags.
- no 1U beam data → `BeamError`.
- non-positive cycle duration → `DataError`.
- no valves operated → `ValveError`.
- sum of period durations exceeds cycle duration → `DataError`.
- 1A current below threshold → `BeamError`.
- 1A current low within 20 s of cycle start (`cycle > 0` branch) → `BeamError`.
- `quiet=True` returns `False` silently; `raise_error=True` raises; default
  prints via `tqdm.write` and returns `False`.

**`shift_timing`:**

- shifts every period start by `dt` and the last period stop by `dt`.

**`draw_cycle_times`** (marker: `plotting`):

- returns the array of non-zero period indices.
- `xmode` in `{datetime, duration_run, duration_cycle, epoch}` all run.
- bad `xmode` raises `RuntimeError`.
- zero-length periods are skipped.
- a filtered-out cycle is drawn struck-through (filter `False` branch).

**`_inspect_draw`:** covered indirectly by `ucnbase.inspect` on a run.

**`get_nhits`:** delegates to `_run._get_nhits` with the cycle index — assert
equals the run-level per-cycle count.

### 6.8 `test_ucnperiod.py` — Layer B + C

**Core:**

- `run[0, 0]` is a `ucnperiod`; `period`, `period_start`, `period_stop`,
  `period_dur` correct.
- period 0 starts at `cycle_start`; period N > 0 starts at the previous
  period's end time.
- `cycle_param.period_durations_s` / `period_end_times` collapsed to scalars
  for this period.
- `tfile` is a `tsubfile` over the period window; `epics` rebound.
- `__repr__` non-empty and contains `cycle`/`period`.
- `get_nhits('Li6')` matches the known per-period count; `bin_ms>0` path.
- `modify_timing(dt_start, dt_stop)` delegates to `_run._modify_ptiming` and
  the period window changes accordingly.

**Edge:**

- a zero-length period (`start == stop`) constructs without error and
  `get_nhits` returns 0.
- `is_pileup('Li6')` returns `False` on clean data and `True` on synthetic data
  with counts above `pileup_cnt_per_ms` within the first `pileup_within_first_s`
  seconds.

### 6.9 `test_chopper.py` — Layer B (`crun`/`ccycle`/`cperiod`/`cframe`)

Use `chopper_file` (`UCNHits_Li6` with channel-15 frame markers).

**Core:**

- `crun(path)` builds; `cycle_param.nframes` and `frame_start_times` set from
  channel-15 timestamps; frames past `cycle_times.stop.max()` dropped.
- `chop_time_ch` configurable (default 15).
- `crun[0]` → `ccycle`; `crun[0, 0]` → `cperiod`; `crun[0, 0, 0]` → `cframe`.
- `crun[:]`, `crun[:, 0]`, `crun[0, :]` slicing returns applylists.
- `crun[1:3, 0, :]` three-level slice.
- `ccycle` trims `frame_start_times` to the cycle window; `nframes` updated.
- `cperiod` trims frames to the period window; `len(cperiod) == nframes`;
  iteration yields `cframe` objects.
- `cframe`: `frame`, `frame_start`, `frame_stop`, `frame_dur`; last frame stops
  at `period_stop`.
- `cframe.get_nhits('Li6')` returns the count for that frame.
- `cframe.__repr__` contains `frame`.
- `crun.get_tof()` returns `(bins, hist)`; histogram length consistent with
  `bin_ms`; `ucn_only` `True` vs `False` branches.
- `crun.get_tof()` excludes filtered (bad) cycles when a filter is set.
- `offset_frames(dt)` shifts all frame start times and updates cached
  cycles/periods.
- `_get_nhits_frame`: run-level returns tree size; per-frame returns the cached
  count.
- `crun.get_cycle()` returns `ccycle` objects (overridden factory).
- `ccycle.get_period()` drops a trailing empty period.

**Edge:**

- `crun[i > ncycles]` / `ccycle[i > nperiods]` / `cperiod[i > nframes]` raise
  `IndexError`.
- bad index type to `crun.__getitem__` / `ccycle.__getitem__` /
  `cperiod.__getitem__` raises `IndexError`.
- `offset_frames` with `dt` pushing frames negative raises `ValueError`.
- `offset_frames` with `dt` pushing frames past run end raises `ValueError`.
- `_get_nhits_frame` with a non-existent frame timestamp raises `KeyError`.
- `cframe` built from a period with `nframes == 0` sets all frame attributes to
  `None` and returns early.

**Plotting** (marker: `plotting`):

- `crun.inspect()` runs, draws alternating frame shading, returns axes.

### 6.10 `test_integration.py` — Layer B (end-to-end)

Workflows mirroring the tutorials, asserting cross-object consistency:

- load run → iterate cycles → iterate periods; counts add up:
  `sum(period.get_nhits)` over a cycle `== cycle.get_nhits`.
- `run.apply(lambda c: c.cycle) == list(range(ncycles))`.
- filter workflow: `gen_cycle_filter` → `set_cycle_filter` → `run[:]` length
  drops by the number of rejected cycles.
- `run[:, 2].get_nhits('Li6')` (applylist element access) equals the per-cycle
  period-2 counts.
- timing workflow: `shift_timing` then re-read `get_nhits` reflects the new
  windows.
- `set_cycle_times` re-run with a different mode updates `cycle_param`
  consistently and cycles still slice correctly.
- datetime round trip on a real tree index pulled from the run.
